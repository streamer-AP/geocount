### Step 3: 解析 GT 标定参数

**任务**：将数据集的 XML 标定文件解析为统一的 numpy 格式。

**输入**：Wildtrack calibrations/ 目录下的 XML 文件

**输出**：每个相机的 3x3 内参矩阵 K 和 3x4 外参矩阵 [R|t]

**关键细节**：
- Wildtrack 存储的是旋转向量（rvec, 3x1），需用 cv2.Rodrigues() 转为旋转矩阵（R, 3x3）
- 外参矩阵 = [R | t]，其中 t 是平移向量（3x1）
- 使用 intrinsic_zero（去畸变后）的内参，因为 VGGT 不输出畸变系数

**核心代码逻辑**：
```python
import cv2
import numpy as np
import xml.etree.ElementTree as ET

def parse_wildtrack_calibration(calib_dir, num_cameras=7):
    """解析 Wildtrack 的 XML 标定文件"""
    cameras = {}
    for cam_id in range(num_cameras):
        # 读取内参（去畸变版本）
        intrinsic_file = f"{calib_dir}/intrinsic_zero/intr_CVLab{cam_id+1}.xml"
        K = parse_opencv_xml_matrix(intrinsic_file)  # 3x3

        # 读取外参
        extrinsic_file = f"{calib_dir}/extrinsic/extr_CVLab{cam_id+1}.xml"
        rvec, tvec = parse_opencv_xml_rvec_tvec(extrinsic_file)
        R, _ = cv2.Rodrigues(rvec)  # 3x1 -> 3x3

        # 构造 3x4 外参矩阵
        extrinsic = np.hstack([R, tvec.reshape(3, 1)])  # [R|t]

        cameras[cam_id] = {
            'intrinsic': K,         # 3x3
            'extrinsic': extrinsic,  # 3x4
            'R': R,                  # 3x3
            't': tvec,               # 3x1
        }
    return cameras
```

### Step 4: 运行 VGGT 推理

**任务**：将多视角同步图像输入 VGGT，获取估计的相机参数。

**输入**：7 张同步图像（Wildtrack C1~C7 的同一帧）

**输出**：每个相机的估计内参和外参

**核心代码逻辑**：
```python
import torch
from vggt.models.vggt import VGGT
from vggt.utils.load_fn import load_and_preprocess_images

def run_vggt_inference(image_paths):
    """用 VGGT 估计多视角相机参数"""
    device = "cuda"
    dtype = torch.bfloat16

    model = VGGT.from_pretrained("facebook/VGGT-1B")
    model = model.to(device).to(dtype).eval()

    # 加载图像 -> (1, N, H, W, 3) tensor
    images = load_and_preprocess_images(image_paths)

    with torch.no_grad():
        outputs = model(images.to(device).to(dtype))

    # 提取相机参数
    extrinsics = outputs["extrinsic"].cpu().numpy()   # (1, N, 3, 4)
    intrinsics = outputs["intrinsic"].cpu().numpy()    # (1, N, 3, 3)

    # 额外输出（后续可能用到）
    depth_maps = outputs["depth"].cpu().numpy()        # (1, N, H, W)
    world_points = outputs["world_points"].cpu().numpy()

    return {
        'extrinsics': extrinsics[0],    # (N, 3, 4)
        'intrinsics': intrinsics[0],    # (N, 3, 3)
        'depth_maps': depth_maps[0],    # (N, H, W)
        'world_points': world_points[0]
    }
```

**注意事项**：
- VGGT 输出的外参是 camera-from-world（OpenCV 惯例），需确认与 GT 一致
- VGGT 的坐标系以第一张图像为参考帧，而 GT 使用场景世界坐标系
  - **这是最关键的对齐问题**，见 Step 5

### Step 5: 评估 — 相机参数精度对比

**任务**：对比 VGGT 估计参数与 GT 参数的差异。

#### 5.1 坐标系对齐（核心难点）

VGGT 输出的相机位姿定义在**自身参考坐标系**中（以某一帧为原点），而 GT 标定参数定义在**场景世界坐标系**中。两者不在同一坐标系下，**不能直接比较绝对值**。

**对齐方法**：使用 Procrustes 分析（Umeyama 算法）

```
目标：找到最优的 Sim(3) 变换 S，使得 S * P_vggt 约等于 P_gt
其中 P_vggt 和 P_gt 分别是 VGGT 和 GT 的相机位置集合
Sim(3) 包含：旋转 R_align, 平移 t_align, 尺度 s
```

```python
def align_poses_sim3(pred_positions, gt_positions):
    """
    Sim(3) 对齐：Umeyama 算法
    pred_positions: (N, 3) VGGT 估计的相机位置
    gt_positions:   (N, 3) GT 相机位置
    返回: 对齐后的 pred_positions, 变换参数
    """
    # 1. 中心化
    pred_mean = pred_positions.mean(axis=0)
    gt_mean = gt_positions.mean(axis=0)
    pred_centered = pred_positions - pred_mean
    gt_centered = gt_positions - gt_mean

    # 2. 计算尺度
    s = np.sqrt((gt_centered ** 2).sum() / (pred_centered ** 2).sum())

    # 3. 计算旋转（SVD）
    H = pred_centered.T @ gt_centered
    U, _, Vt = np.linalg.svd(H)
    R = Vt.T @ U.T
    if np.linalg.det(R) < 0:
        Vt[-1, :] *= -1
        R = Vt.T @ U.T

    # 4. 计算平移
    t = gt_mean - s * R @ pred_mean

    # 5. 对齐
    aligned = s * (R @ pred_positions.T).T + t
    return aligned, {'R': R, 't': t, 's': s}
```

#### 5.2 评估指标

| 指标 | 计算方法 | 含义 |
|-----|---------|------|
| **相机位置误差** | 对齐后的欧氏距离 | 米，反映平移精度 |
| **旋转角度误差** | arccos((tr(R_pred * R_gt^T) - 1) / 2) | 度，反映朝向精度 |
| **相对旋转误差** | 任意两相机间相对旋转的误差 | 度，不受坐标系影响 |
| **相对平移角度误差** | 两相机间平移方向的角度差 | 度，不受尺度影响 |
| **重投影误差** | 用估计参数投影 GT 3D 点到图像 | 像素，最直观 |
| **内参相对误差** | 焦距差 / GT焦距 | 百分比 |

#### 5.3 重投影误差计算（最重要的指标）

```python
def compute_reprojection_error(gt_3d_points, K_pred, R_pred, t_pred,
                                K_gt, R_gt, t_gt):
    """
    计算重投影误差
    gt_3d_points: (M, 3) 地面平面上的 GT 3D 点
    """
    # 用 GT 参数投影
    proj_gt = K_gt @ (R_gt @ gt_3d_points.T + t_gt.reshape(3,1))
    proj_gt = proj_gt[:2] / proj_gt[2:]

    # 用估计参数投影（对齐后）
    proj_pred = K_pred @ (R_pred @ gt_3d_points.T + t_pred.reshape(3,1))
    proj_pred = proj_pred[:2] / proj_pred[2:]

    # 误差
    error = np.linalg.norm(proj_pred - proj_gt, axis=0)
    return error.mean(), error
```

#### 5.4 可行性判断标准

| 指标 | 优秀 | 可用 | 需改进 |
|-----|------|------|-------|
| 旋转误差 | < 2度 | 2~5度 | > 5度 |
| 位置误差（对齐后） | < 0.2m | 0.2~0.5m | > 0.5m |
| 重投影误差 | < 10px | 10~30px | > 30px |
| 内参误差 | < 5% | 5~15% | > 15% |

## 潜在问题与预案

| 问题 | 预案 |
|-----|------|
| VGGT 输入图像被 resize 后丢失监控场景细节 | 检查 VGGT 的预处理流程，必要时分块推理 |
| 坐标系对齐后误差仍然很大 | 检查是否有镜像/轴交换问题；尝试枚举坐标系惯例 |
| 监控视角间 baseline 太大，VGGT 匹配困难 | 尝试增加中间视角、或只用相邻视角子集 |
| GT 标定本身有误差 | 用重投影误差交叉验证 GT 质量 |
